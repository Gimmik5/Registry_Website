## Setup: get the webhook signing secret

Webhooks need their own signing secret, generated by `stripe listen`. We need this BEFORE the dev server starts so it can verify incoming events.

### Terminal 1 â€” start the Stripe webhook tunnel
```
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

You'll see output ending with:
```
> Ready! You are using Stripe API Version [...]. Your webhook signing secret is whsec_AbCdEf...
```

**Copy the `whsec_...` value.**

### Edit `.env.local`
Paste the secret:
```
STRIPE_WEBHOOK_SECRET="whsec_AbCdEf..."
```

### Terminal 2 â€” start the dev server
```
npm run dev
```

(Restart it if it was already running â€” `.env.local` only loads at startup.)

You should now have:
- **Terminal 1:** `stripe listen` running, ready to forward events
- **Terminal 2:** Next.js dev server at http://localhost:3000

## Common failure modes

| Symptom | Likely cause | Fix |
|---|---|---|
| Webhook returns 400, `signature verification failed` | `STRIPE_WEBHOOK_SECRET` doesn't match the secret printed by `stripe listen` | Re-copy from terminal 1, restart dev server |
| `stripe listen` shows `[400]` | Same as above, OR dev server isn't running, OR `proxy.ts` is intercepting `/api/webhooks/stripe` (it shouldn't â€” webhook path isn't in the matcher) | Check terminal 2 output for the actual error |
| Stripe redirect lands on `/checkout/success` but gift isn't updated | Webhook didn't fire OR fired but errored. Check terminal 1 for the event status. | Look at dev server logs in terminal 2 for the error stack. Common cause: DATABASE_URL not loaded. |
| `Purchase` row doesn't exist | `/api/checkout` failed before creating it. Check dev server logs. | Likely a Stripe API error â€” check the `STRIPE_SECRET_KEY` is valid. |
| Port 3000 already in use | Another dev server is running | `npm run dev -- -p 3001` and update `stripe listen --forward-to localhost:3001/...` |
| Group gift accepts more than the remaining amount | Server validation in `/api/checkout` should catch this | Check `src/app/api/checkout/route.ts` group-gift branch. The form clamping is a UX nicety; the server enforces it. |
| Fund auto-marks as PURCHASED | Bug in `incrementGroupGiftRaised` â€” should skip the close logic when `isFund: true` | Check `src/lib/database/gifts.ts` |
